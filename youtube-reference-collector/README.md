# 유튜브 레퍼런스 수집기

주제어(키워드)를 입력하면 관련 유튜브 영상을 모아, **채널 평균 대비 잘 터진 영상(아웃라이어)** 만
골라 보여주는 단일 실행형 CLI 도구입니다.

> **핵심 가설** — 한 영상의 조회수가 그 채널의 평균 조회수보다 훨씬 높다면(아웃라이어),
> 그건 채널 규모 덕분이 아니라 그 영상의 **주제·제목·썸네일이 먹혔다**는 뜻입니다.
> 즉 **베껴올 가치가 있는 레퍼런스**입니다. 이 도구는 그걸 자동으로 찾아냅니다.

---

## 무엇을 하나요

1. YouTube Data API v3 로 주제어를 검색합니다.
2. 각 영상의 조회수·좋아요·댓글·업로드일·길이·채널·구독자를 모읍니다.
3. 각 영상이 속한 채널의 **최근 영상 평균 조회수**를 계산합니다.
4. `영상 조회수 ≥ 채널 평균 × 4`(기본값) 인 **아웃라이어만** 남깁니다.
5. 조회수 높은 순으로 정렬해 콘솔 표 + CSV/JSON 으로 저장합니다.

---

## 설치

```bash
cd youtube-reference-collector
python -m venv .venv && source .venv/bin/activate   # (선택) 가상환경
pip install -r requirements.txt
```

의존성은 `google-api-python-client` 와 `python-dotenv` 두 개뿐이며,
CSV/JSON 저장은 파이썬 표준 라이브러리만 사용합니다(별도 pandas 불필요).

모든 기능(검색 → 통계 수집 → 채널 평균 → 필터/정렬 → 출력)은 **`collect.py` 파일 하나**에
들어 있습니다. `chmod +x collect.py` 후 `./collect.py "주제어"` 로 바로 실행할 수도 있습니다.

## API 키 발급 방법

1. [Google Cloud Console](https://console.cloud.google.com/) 접속 → 새 프로젝트 생성.
2. **API 및 서비스 → 라이브러리** 에서 **YouTube Data API v3** 를 검색해 **사용 설정**.
3. **API 및 서비스 → 사용자 인증 정보 → 사용자 인증 정보 만들기 → API 키**.
4. 생성된 키를 복사합니다. (선택: 키 제한에서 YouTube Data API v3 만 허용)

발급받은 키는 코드에 하드코딩하지 말고 환경변수/`.env` 로 넣습니다.

```bash
# 방법 A: 환경변수
export YOUTUBE_API_KEY='발급받은_키'

# 방법 B: .env 파일
cp .env.example .env      # 그리고 .env 안의 값을 실제 키로 교체
```

## 실행

```bash
# 가장 단순한 사용
python collect.py "퇴사"

# 옵션 조합
python collect.py "재테크 초보" --multiplier 5 --since 6m --sort velocity
python collect.py "홈카페" --max-results 200 --sample 40 --order viewCount
```

### 옵션

| 옵션 | 기본값 | 설명 |
|---|---|---|
| `keyword` | (필수) | 검색할 주제어 |
| `--multiplier` | `4` | 아웃라이어 배수 임계값 (조회수 ≥ 채널 평균 × 이 값) |
| `--max-results` | `150` | 검색으로 모을 후보 영상 수 |
| `--sample` | `30` | 채널 평균 계산에 쓸 최근 영상 표본 수 |
| `--since` | 없음 | 업로드 기간 필터 (`6m`, `1y`, `90d` 등) |
| `--sort` | `views` | 결과 정렬: `views` / `velocity` / `multiplier` |
| `--order` | `relevance` | 검색 정렬: `relevance` / `viewCount` / `date` |
| `--out-dir` | `.` | 결과 파일 저장 폴더 |

결과 파일: `references_{주제어}_{날짜}.csv` 와 `.json` (썸네일 URL 포함).

## 결과 컬럼 / 지표

| 지표 | 의미 |
|---|---|
| **아웃라이어 배수**(핵심) | 영상 조회수 ÷ 채널 평균(같은 형식). 평균·중앙값 둘 다 계산 |
| 일일 조회수(velocity) | 조회수 ÷ 업로드 후 경과일 → 최근 가속 중인 영상 포착 |
| 참여율 | (좋아요 + 댓글) ÷ 조회수 → 낚시 클릭과 진짜 반응 구분 |
| 구독자 대비 조회수 | 조회수 ÷ 구독자 수 → 작은 채널의 아웃라이어 강조 |
| 제목 패턴 | 질문형/숫자형/자극·호기심형/정보·하우투형 등으로 분류 |

### 정확도를 위한 계산 원칙

- **최근 영상 기준 평균**: 채널 전체가 아니라 최근 N개(기본 30개)로 평균을 냅니다.
- **중앙값 병행**: 한 영상의 떡상으로 평균이 부풀려지는 문제를 완화합니다.
- **숏폼/롱폼 분리**: 60초 이하 숏폼과 롱폼은 조회수 스케일이 달라
  **같은 형식끼리만** 비교합니다.
- **신뢰도 표시**: 같은 형식 표본이 5개 미만이면 결과에 `⚠`(신뢰도 낮음)로 표시합니다.
- **비공개 구분**: 좋아요/댓글/구독자가 비공개면 `0` 이 아니라 `비공개`(null)로 구분합니다.

## 레퍼런스를 "잘" 모으기 위한 운영 원칙

1. **아웃라이어 배수 > 채널 규모.** 조회수 100만이어도 채널 평균이 80만이면 평범한 영상,
   조회수 5만이어도 평균이 5천이면 10배 터진 진짜 레퍼런스입니다.
2. **작은 채널의 아웃라이어를 놓치지 마세요.** 구독자 후광 없이 주제·제목·썸네일로
   이긴 케이스라 가장 베낄 가치가 큽니다. (`구독자 대비 조회수`로 확인)
3. **최근성을 봅니다.** `--since` 로 최근 3~6개월 영상만 보면 "지금 통하는 수요"가 보입니다.
4. **velocity 로 지금 뜨는 영상을 잡습니다.** 누적 조회수는 오래된 영상에 유리합니다.
5. **제목·썸네일이 진짜 상품입니다.** 둘 다 수집하며, 결과를 제목 패턴별로 묶어 보여줍니다.
6. **참여율로 진짜 반응을 검증합니다.** 조회수만 높고 참여율이 낮으면 낚시일 수 있습니다.
7. **수집은 분류로 이어집니다.** 콘솔 하단에 어떤 앵글이 반복적으로 터지는지 요약합니다.

## API 쿼터 주의

- YouTube Data API 기본 일일 쿼터는 **10,000 유닛**입니다.
- `search.list` 는 **1회당 100 유닛**으로 비쌉니다. `videos.list`/`channels.list`/
  `playlistItems.list` 는 각 **1 유닛**입니다.
- 이 도구는 검색을 필요한 만큼만 하고, 상세 조회는 항상 **50개씩 배치**로 묶어 쿼터를 아낍니다.
- 실행이 끝나면 **총 소비 쿼터**를 출력합니다.
- 쿼터를 초과하면 명확한 메시지와 함께 **그때까지 수집한 부분 결과를 저장**합니다.

### 대략적인 쿼터 소비 예시

`--max-results 150` (검색 3페이지) + 채널 50개 평균 계산 기준:
검색 300 + 영상/채널/재생목록 상세 약 150~200 = **대략 450~500 유닛** 수준입니다.
(후보 수·채널 수에 따라 달라집니다.)
